# ADR-010: Аварийное переключение (Failover) и Защита периметра

## Статус
Принято

## Контекст
При SLA 99.99% платформе GoFuture разрешено "лежать" не более 52 минут за весь год. Платформа обязана выживать при отключении электропитания дата-центров (Availability Zones) и даже при падении региона целиком (землетрясения/уязвимости гипервизора), а также автоматически отражать кибератаки (DDoS-штормы 7-го уровня, скрапинг).

## Решение по Безопасности (Security Posture)

Для отражения атак еще до того, как они дойдут до наших микросервисов, мы разворачиваем эшелонированную сеть защиты на Edge-локациях (на границе входа трафика).

1. **Network Firewall / Anti-DDoS (уровень L3/L4):** Отражает SYN floods, UDP reflection и другие атаки на переполнение канала. Инструменты (например AWS Shield) автоматически отбрасывают мусорный трафик в Blackhole.
2. **WAF (Web Application Firewall, уровень L7):** Аналитическая пред-фильтрация всех HTTP/HTTPS пакетов.
   *Обоснование внедрения:* Необходим для защиты API от попыток SQL-инъекций, эксплойтов OWASP Top-10 и Cross-Site Scripting. Любые "грязные" запросы получают бан по IP. Обновление правил WAF блокирует 0-day уязвимости до того, как мы успеем пропатчить приложения.
3. **API Rate Limiting:** Устанавливается на Gateway для отбития ботов (перебор промокодов, парсинг геодата водителей конкурентами).

---

## Решение по Отказоустойчивости (Failover)

Высокая доступность достигается за счет дублирования:
- **Intra-Region (Доменные сбои):** Использование минимум 3-х Availability Zones (Изолированных зданий ЦОД) внутри одного региона (например, в Сингапуре). Сбой одного здания не останавливает регион.
- **Cross-Region (Глобальные сбои):** Сбой целого региона обрабатывается через Global Routing Layer.
